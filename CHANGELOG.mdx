# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Prefix the change with one of these keywords:

- _Added_: for new features.`
- _Breaking_: for higher visibility of breaking changes
- _Changed_: for changes in existing functionality.
- _Deprecated_: for soon-to-be removed features.
- _Removed_: for now removed features.
- _Fixed_: for any bug fixes.
- _Security_: in case of vulnerabilities.

## [Unreleased]

- _Added_: Dark color scheme pattern via `data-color-scheme="dark"` attribute (`src/styles/base/_color-scheme.scss`). Apply to any element — including `<html>` for full-page dark mode — to flip descendant text, headings, and bare links to light colours. `WideWave` now applies this automatically since both its `black` and `red` backgrounds are always dark.
- _Added_: `contentWidth` prop to `Section` — wraps children in a constrained inner `has-global-padding` div so the background width (`maxWidth`) and content column width can be set independently; `false` = 1024px, `true` = 1280px.
- _Added_: `--rds--gradient-white-to-pale-grey` design token — subtle light gradient for use as a section background.
- _Added_: `postHeader` prop to `PageHeader` — renders an italic light-weight line below the heading (e.g. a subtitle or date range); styled via `.cu-pageheader__post`.
- _Added_: `maxWidth` prop to `Column` — accepts `aligncontent | alignwide | alignfull`; defaults to `aligncontent`. Also adds `cu-layout` to the root class list so layout spacing utilities apply.
- _Added_: `UnorderedList` and `OrderedList` helpers to `src/data/storyContent.tsx` (replaces `SampleList`); both include nested lists and inline links for richer story demos.
- _Changed_: `ImageCover` rebuilt — `maxWidth` and `contentWidth` props align with `Section`; `image` prop accepts a name key (e.g. `"tory"`) and constructs the CDN URL internally (`https://cdn.carleton.ca/rds/assets/bg-images/{name}.jpg`); `opacity` (0–100, default 85) controls the white overlay via `::after`. Background image styles are scoped to a `cu-imagecover--image` modifier so the grey background always shows when no image is provided. Removed `focalPointX`/`focalPointY`; background defaults to `50% 50%`. Wave and overlay use `isolation: isolate` + negative z-index so content is always above them without explicit z-index. Added `BgImagesData.json` as the image name source.
- _Changed_: `Section` — `isGrey?: boolean` replaced by `bgType?: 'grey' | 'black' | 'light-gradient'`. Update all usages: `isGrey` → `bgType="grey"`. `bgType="black"` automatically adds `data-color-scheme="dark"` for accessible text inversion.
- _Changed_: `FullBanner` root class now includes `cu-layout` so layout spacing utilities apply. `PageHeader` inside `FullBanner` always uses `size="md"` regardless of heading level.
- _Changed_: Layout spacing selectors in `_utils.scss` tightened — gap rules are now scoped to direct children of `.cu-main > *` to prevent bleed into nested contexts; adjacent `alignfull` layout siblings collapse their margin gap; first `alignfull` inside `.cu-main--padding` pulls up via negative margin.
- _Changed_: `PageHeader` heading-to-content gap reduced (`margin-top` on `& + p` changed from `spacing-medium` to `spacing-small`); underline variant gains a bottom margin to create breathing room between the underline and following content.
- _Added_: `StackedList` component; bordered card-style container for arranging list items in a one- or two-column responsive grid with optional section heading and shadow control.
- _Fixed_: `WideWave` red variant background changed from `--rds--color-primary` to `--rds--color-primary-dark` to meet WCAG AA color contrast with white text.
- _Added_: `Embed` component — responsive 16:9 wrapper for YouTube, Vimeo, Kaltura, TED, SoundCloud, Audioboom, and Power BI iframes, plus standalone `EmbedHubSpot` for HubSpot forms. Replaces Tailwind-based `isRounded` with `maxWidth: aligncontent | alignwide | alignfull` (same alignment system as `Section`). Border-radius always applied via `--rds--radius-md`; aspect-ratio via CSS `aspect-ratio: 16 / 9`.

## [0.5.11] - 2026-05-27

- _Added_: `Figure` component — semantic `<figure>` wrapper for images and media with an optional `caption` (rendered in `<figcaption>` with a grey-pale background band). `size` prop (`xs` 320px · `sm` 440px · `md` 512px · `lg` 48rem · `full` 100%) sets an explicit width on the root so the inner wrapper and caption are always bounded to the same width as the image. `align` prop (`none` · `left` · `right` · `center`) enables float-based text-wrapping layouts; floats activate at the `md` breakpoint (784px) and the figure collapses to full width below it. Images are always `width: 100%; height: auto` so they scale with the container at every viewport width. When `align="left"` or `align="right"`, the component adds `alignleft`/`alignright` to the root element to opt out of the `is-layout-constrained` `margin: auto !important` rule in `layout.css` — without this the float margins would be suppressed. Border radius (`--rds--radius-md`) and `overflow: hidden` are applied to the inner wrapper so the image corners are clipped correctly.
- _Changed_: Updated `@troychaplin/component2block` to `^0.5.9` and added `"emitScssAlongside": true` to `c2b.config.json`. The build now generates `.scss` copies of `tokens.css`, `base-styles.css`, `layout.css`, and `typography.css` alongside the `.css` originals in both `src/styles/auto/` and `dist/cutheme/`. This enables Dart Sass consumers to `@use "@cuweb/rds2/dist/cutheme/tokens"` (without the `.css` extension) and have the file inlined correctly — Dart Sass treats `@use` on a `.css`-suffixed path as a plain CSS passthrough rather than importing the file.
- _Fixed_: `build-components-dts.mjs` no longer fails with TS7016 ("Could not find a declaration file for module '../../styles/auto/tokens'"). `@troychaplin/component2block` v0.5.9 now generates a `tokens.d.ts` declaration file alongside `tokens.js` in both `srcDir` and `outputDir`, providing the type information TypeScript requires when `src/components/tokens/index.ts` re-exports from the auto-generated module.

## [0.5.9] - 2026-05-25

- _Added_: `CallOut` component — a bordered content block for drawing attention to key messages and CTAs. Wraps a `PageHeader` (`size="md"`, underline on by default) followed by arbitrary children (typically a paragraph and `ButtonGroup`). Supports `justify` (`start`, `center`, `end`) which aligns text, the accent underline, and any nested `ButtonGroup` in one prop; and `maxWidth` (`aligncontent`, `alignwide`, `alignfull`) to constrain the box width within the layout grid. Centered layout is handled via `isCenter` on the inner `PageHeader`; end-justified underline repositioning is done via scoped SCSS override.
- _Added_: `SectionsLayout` Storybook template (`Overview/Templates/Sections Layout`) — fullscreen reference story composing `Nav`, `Main`, `PageHeader`, `CallOut`, `Section` (white, grey, alignwide, alignfull variants), typography samples, `FooterStandard`, and `CookieBanner` to validate real-page layout and component interaction.
- _Added_: Granular CSS export paths `./base-styles.css`, `./layouts.css`, and `./typography.css` to `package.json` exports map — exposes individual style partials from `dist/cutheme/` for consumers that need a subset of design system styles without importing the full `dist/style.css`.
- _Fixed_: `Section` first-child margin removal selector scoped from `.cu-main .alignfull:first-child` to `.cu-main .cu-section--grey.alignfull:first-child` — prevents inadvertently stripping top margin from other full-width elements that are not grey sections.

## [0.5.8] - 2026-05-25

- _Added_: `Nav` compound component (`Nav` with `Nav.Top`, `Nav.Bottom`, `Nav.Logo`, `Nav.Menu`, `Nav.Buttons`) — a sticky site header whose primary menu uses a priority-plus pattern: it measures the available width and overflows items into a "Browse" dropdown instead of wrapping. On scroll down the white top row (logo + site title) slides up out of view while a `Nav.Bottom` strip stays pinned to the top; scrolling up brings the top row back. (With no bottom strip, the whole header hides/reveals.)
- _Added_: Responsive mobile layout for `Nav` — below the `sm` breakpoint (600px) the logo takes its own row and the menu collapses to a single Browse trigger (which opens the full menu) beside the search/CTA buttons in a full-width grey bar. Applies to both the top-composed menu and the `Nav.Bottom` composition; the wrapper relocates the buttons into the bottom bar on mobile only, leaving the desktop layout untouched.
- _Added_: `variant` prop on `Nav.Buttons` items (`'primary'` red default, `'dark'` black) for secondary CTAs such as Donate.
- _Added_: `PageHeader` component — semantic `<header>` block for page and section titles. Supports five size variants (`sm`, `md`, `lg`, `xl`, `primary`) that map directly to the c2b heading token scale; content paragraph font size auto-scales with the heading size without a separate prop. Optional `preHeader` label, `pronoun` (appended in parentheses), `metaData` slot, and `content` string (truncated at 320 characters). Includes `isWhite` for dark backgrounds, `isCenter` for centred layouts, and `noUnderline` (on by default) to opt into the red accent underline.
- _Added_: `Avatar` component — displays a user photo or initials fallback in `sm`, `md`, and `lg` sizes with square or circle shape. Renders as a `<button>` when `onClick` is provided for accessible interactive use.
- _Added_: `SocialIcons` compound component (`SocialIcons` + `SocialIcons.Item`) — icon-only links using `IconName` from `@cuweb/rds-icons`, open to any icon rather than a hardcoded service registry. Icons size responsively via `font-size` on the link. `orcid` icon pending addition to `@cuweb/rds-icons`.
- _Added_: `ProgressBar` component — accessible progress bar with required `label` prop for `aria-label`, `value`/`max` props with internal percentage clamping. Fill uses `position: absolute` to avoid percentage height resolution issues. Storybook stories use a fixed-width decorator so args-based stories render correctly in Storybook's flex canvas.

## [0.5.3] - 2026-05-22

- _Added_: `./tokens.css` export path — exposes CSS custom properties only (no `modern-normalize`); intended for Ellucian Experience and other platforms that inject CSS globally where the full `dist/style.css` would conflict with the host design system.
- _Added_: `scripts/build-components-dts.mjs` — standalone TypeScript declaration bundler for per-component builds. Runs `tsc` to emit declarations then uses `rollup-plugin-dts` to bundle each component's declaration chain into a self-contained `index.d.ts`. Replaces `vite-plugin-dts` in the per-component build, which was producing empty `export {}` stubs due to a path resolution conflict between `outDir` and TypeScript's inferred `rootDir`.
- _Added_: `rollup` and `rollup-plugin-dts` as direct `devDependencies` — Vite 8 uses rolldown internally so `rollup` is no longer a transitive dependency and must be declared explicitly.
- _Changed_: `@cuweb/rds-icons` is now externalized across all build targets — it was previously compiled inline. Declared in `peerDependencies` with `optional: true` in `peerDependenciesMeta`. Root bundle reduced from ~316 kB to ~103 kB.
- _Changed_: Per-component JS build (`vite.config.components.ts`) now auto-discovers components from `src/components/` via `readdirSync` — no hardcoded name list. `Card`, `Status`, and `Icon` were previously absent from `dist/components/`.
- _Changed_: Updated `c2b.config.json` — renamed config key `themeDir` to `outputDir` to match `@troychaplin/component2block` v0.5.6.
- _Fixed_: Per-component `index.d.ts` files were empty stubs (`export {}`) — caused by `vite-plugin-dts` discarding compiled declarations whose output paths fell outside the configured `outDir`. Resolved by replacing the plugin with `scripts/build-components-dts.mjs`.
- _Fixed_: `dropRootCss` Vite plugin was silently failing — it was attempting to delete `dist/components/rds-2.0.css` but the actual generated file is `rds2.css`.
- _Removed_: Top-level `main`, `module`, and `types` fields from `package.json` — these are superseded by the `exports` map and were redundant.
- _Removed_: `./*.scss` entry from the `exports` map — per-component SCSS source files are not published artifacts.
