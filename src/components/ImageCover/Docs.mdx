# ImageCover

A page-level content container with a decorative background image, white overlay, and red wave along the bottom edge. Structurally identical to `Section` — same `maxWidth`, `isGrey`, and `contentWidth` props — with `image` and `opacity` added to control the background layer.

## When to use

- Adding branded visual impact to a page section without sacrificing content readability.
- Full-bleed (`alignfull`) hero or accent bands where the image spans edge-to-edge.
- Constrained variants (`aligncontent` / `alignwide`) for mid-page visual breaks that respect the layout grid.
- Use `contentWidth` alongside `maxWidth="alignfull"` to keep the background full-bleed while constraining child content to a readable column.

## Import

```tsx
import { ImageCover } from '@cuweb/rds2';
```

## Basic usage

```tsx
// Default — content width, image background with grey tint
<Main>
  <ImageCover image="https://example.com/photo.jpg" isGrey>
    <Column cols="2">...</Column>
  </ImageCover>
</Main>

// Full-bleed image band
<Main>
  <ImageCover image="https://example.com/photo.jpg" isGrey maxWidth="alignfull">
    <Column cols="3">...</Column>
  </ImageCover>
</Main>

// Full-bleed image, content constrained to 1024px
<Main>
  <ImageCover image="https://example.com/photo.jpg" isGrey maxWidth="alignfull" contentWidth={false}>
    <Column cols="2">...</Column>
  </ImageCover>
</Main>

// Full-bleed image, content constrained to 1280px
<Main>
  <ImageCover image="https://example.com/photo.jpg" isGrey maxWidth="alignfull" contentWidth>
    <Column cols="2">...</Column>
  </ImageCover>
</Main>

// Reduced opacity — more of the image shows through
<Main>
  <ImageCover image="https://example.com/photo.jpg" isGrey opacity={60}>
    <Column cols="2">...</Column>
  </ImageCover>
</Main>
```

## Max-width presets

| `maxWidth` | Token | Description |
|---|---|---|
| `"aligncontent"` | `--rds--layout-content-size` | Default — standard readable content width |
| `"alignwide"` | `--rds--layout-wide-size` | Wider — for card grids and media-rich sections |
| `"alignfull"` | `--rds--layout-full-size` | Full viewport width — hero and accent bands |

## The `as` prop

By default `ImageCover` renders a `<section>` element. Pass `as="div"` when a section landmark is semantically inappropriate.

## The `contentWidth` prop

When set, `contentWidth` wraps all children in an inner `has-global-padding` div constrained within the outer layout:

| Value | Inner content width |
|---|---|
| `false` | 1024px (`aligncontent`) |
| `true` / `contentWidth` (no value) | 1280px (`alignwide`) |

Omitting `contentWidth` leaves children as direct descendants of the root.

## CSS classes

| Class | When applied |
|---|---|
| `cu-imagecover` | Always |
| `cu-imagecover--white` | `isGrey` is false (default) |
| `cu-imagecover--grey` | `isGrey={true}` |
| `aligncontent` | `maxWidth="aligncontent"` (default) |
| `alignwide` | `maxWidth="alignwide"` |
| `alignfull` | `maxWidth="alignfull"` |
| `has-global-padding` | When `isGrey={true}` — horizontal gutter |
| `is-layout-constrained` | Always — constrains children to content / wide widths |
| `cu-imagecover__wave` | Always — decorative wave (`alignfull` to opt out of layout constraint) |
| `cu-imagecover__overlay` | Always — white opacity layer (`alignfull` to opt out of layout constraint) |

## Design tokens

| Token | Used for |
|---|---|
| `--rds--color-grey-pale` | Grey background when `isGrey={true}` |
| `--rds--spacing-large` | Padding on grey variant |
| `--rds--spacing-2-x-large` | Bottom padding — creates space above the wave |
| `--rds--radius-xl` | Border radius on non-full-width grey variant |
| `--rds--asset-cu-waves-repeating-bottom-red` | Decorative wave pattern |
| `--rds--layout-content-size` | `aligncontent` max-width |
| `--rds--layout-wide-size` | `alignwide` max-width |
| `--rds--layout-full-size` | `alignfull` max-width (100%) |

### CSS custom properties (set inline per instance)

| Property | Default | Controls |
|---|---|---|
| `--cu-imagecover-bg` | `none` | Background image URL (set via `image` prop) |
| `--cu-imagecover-overlay` | `rgba(255,255,255,0.85)` | White overlay colour/opacity (set via `opacity` prop) |

## Accessibility

- The component renders as a `<section>` landmark; give it an accessible name with `aria-label` or an in-content heading so screen readers can identify the region.
- The background image is purely decorative and applied via CSS — all meaningful content must be expressed through children.
- The wave and overlay carry `aria-hidden="true"` so they are invisible to assistive technology.
- Use `isGrey` together with `image` to ensure the grey tint is present; lower `opacity` values reduce the white overlay and may decrease text contrast.
- Interactive elements inside `children` receive standard keyboard focus and are unaffected by the decorative layers.
