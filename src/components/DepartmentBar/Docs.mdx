# Department Bar

A compact contact strip placed directly above the site footer. Displays a department name, location, phone, and email — with optional call-to-action buttons that push to the trailing edge on desktop.

## When to use

Place `DepartmentBar` once per page, between the main content and the `Footer`. It is most useful on department and unit pages where users need quick access to contact information without scrolling to the footer.

## Import

```tsx
import { DepartmentBar } from '@cuweb/rds2';
```

## Basic usage

```tsx
// Full department contact bar with CTA buttons
<DepartmentBar
  deptName="Information Technology Services"
  officeNumber="400"
  buildingName="Pigiarvik (ᐱᒋᐊᕐᕕᒃ)"
  phone="613-520-2600"
  email="noreply@carleton.ca"
  buttons={[
    { id: 1, title: 'Contact Us', url: 'https://carleton.ca/about/contact/' },
    { id: 2, title: 'Hours', url: '#' },
  ]}
/>

// Contact info only — no buttons
<DepartmentBar
  deptName="Information Technology Services"
  phone="613-520-2600"
  email="noreply@carleton.ca"
/>
```

## Button behaviour

When `buttons` is provided, the first button renders with `color="red"` and all subsequent buttons with `color="dark-grey"`. Buttons are wrapped in a `ButtonGroup align="end"` and push to the trailing edge of the bar on desktop (`md` and above).

```tsx
import { defaultFooterButtons } from '@cuweb/rds2/data/FooterData';

<DepartmentBar buttons={defaultFooterButtons} />
```

## Email link

The `email` prop is automatically rendered as a `mailto:` link via the `LinkProvider` context. If your application uses a framework-specific `<Link>` component, wrap the app root with `LinkProvider` and the email link will adopt it automatically.

## Meta item separators

On desktop (`lg` breakpoint and above), the contact meta items (location, phone, email) are displayed inline with a left-border pipe separator between them. On smaller screens they stack vertically, centred.

## Accessibility

- `DepartmentBar` renders as `<aside>`, an ARIA landmark. Screen reader users can navigate to it directly via landmark navigation.
- The department name is rendered as an `<h2>`. Ensure this fits the page heading hierarchy — if the page already has section headings at `<h2>`, consider whether a visual label rather than a heading is more appropriate.
- The email link is a native `<a href="mailto:...">` and is fully keyboard accessible.
- CTA buttons inherit all accessibility behaviour from the `Button` component.
