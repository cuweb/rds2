# FilterPanel

A composable filter and sort control bar that manages its own state via React context. Renders a top bar with sort and filter dropdowns and, when filters are active, a bottom bar displaying removable filter tags.

## When to use

- Above a card grid, listing, or table where users need to narrow results by category, type, or other attributes.
- When results should support sorting as well as filtering (sort and filters are both optional).
- When the filter state needs to be communicated back to the parent (e.g. to fetch filtered data from an API) via the `onFilterChange` callback.
- Do not use when filtering is handled server-side without any client interaction — render results directly instead.

## Import

```tsx
import { FilterPanel } from '@cuweb/raven-design-system';
import type { SortOption, FilterGroup, ActiveFilters } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
const sortOptions: SortOption[] = [
    { label: 'Newest', value: 'newest' },
    { label: 'Oldest', value: 'oldest' },
];

const filterOptions: FilterGroup[] = [
    {
        id: 'category',
        name: 'Category',
        options: [
            { value: 'design', label: 'Design' },
            { value: 'development', label: 'Development' },
            { value: 'accessibility', label: 'Accessibility' },
        ],
    },
];

const handleFilterChange = (filters: ActiveFilters) => {
    console.log(filters.sortBy, filters.filterBy);
};

<FilterPanel onFilterChange={handleFilterChange}>
    <FilterPanel.Top sortOptions={sortOptions} filterOptions={filterOptions} />
    <FilterPanel.Bottom />
</FilterPanel>;
```

`FilterPanel.Bottom` renders nothing when no filters are active — it is safe to always include it.

| `cu-filter-panel__sort-item--selected`   | The currently active sort option                       |

| `cu-filter-panel__tag`                   | An individual active filter tag pill                   |
| `cu-filter-panel__tag-remove`            | The × button inside an active filter tag               |

## Accessibility

- Sort and filter toggle buttons use `aria-expanded` (boolean) to communicate open/closed state to assistive technology.
- The sort dropdown carries `aria-haspopup="listbox"` and its menu has `role="listbox"` with `role="option"` and `aria-selected` on each item.
- Filter group menus use `aria-controls` to link each toggle to its popup; `aria-labelledby` links the menu back to its trigger.
- Active filter count badges include a visually hidden `aria-label` (e.g. "2 selected") so screen readers announce the count in context.
- Each tag remove button has an `aria-label` in the form `Remove filter: [label text]` so it is unambiguous when read in isolation.
- Pressing `Escape` closes any open dropdown. Clicking outside the top bar also closes the open dropdown.
- Checkboxes use native `<input type="checkbox">` elements with associated `<label>` via `for`/`id` — keyboard-operable without custom ARIA roles.
