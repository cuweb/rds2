# Calendar

An interactive month-view calendar for selecting a single date or a date range. Optionally renders event indicator dots on days that have associated events.

## When to use

- Use `mode="single"` to let a user pick one date — for example, filtering an event listing to a specific day.
- Use `mode="range"` to let a user pick a start and end date — for example, filtering events across a span of days.
- Pass `events` to surface visual indicators (dots) on days with associated events so users can scan availability at a glance.
- Pair with an event listing component in a side-by-side column layout for a full event browser pattern.

## Import

```tsx
import { Calendar } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
// Single-day picker
<Calendar
  mode="single"
  events={events}
  onSelect={(value) => console.log(value)}
/>

// Date-range picker
<Calendar
  mode="range"
  events={events}
  onSelect={(value) => {
    if (Array.isArray(value)) {
      const [start, end] = value;
    }
  }}
/>

// Pre-select a date
<Calendar
  mode="single"
  defaultDate="2025-09-15"
  onSelect={(value) => console.log(value)}
/>
```

## Accessibility

- The day grid uses `role="grid"` with `aria-label` set to the current month and year so screen readers announce context when entering the grid.
- Each day cell has `role="gridcell"`. Each day button uses `aria-pressed` to communicate selected state to assistive technology.
- Weekday labels are wrapped in a container with `aria-hidden="true"` — they are presentational abbreviations; the `<time dateTime="yyyy-MM-dd">` element on each button conveys the full date to screen readers.
- Prev/next buttons use `aria-label="Previous month"` and `aria-label="Next month"` rather than relying on icon-only content.
- Past dates are rendered as `disabled` buttons — they are excluded from the tab order and announced as unavailable.
- Event dots use `aria-hidden="true"` — they are visual cues only. Do not rely on dots alone to communicate event presence; pair the calendar with a visible event listing.
- Color is not the only signal for selection: `aria-pressed` carries the same state for users who cannot perceive color.
