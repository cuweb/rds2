# Badge

A small inline label used to categorize content, indicate status, or tag items with metadata. Renders as a `<span>` by default, or as a link when an `href` is provided.

## When to use

Use `Badge` to surface a short, scannable label — a category tag, a content type, a semantic state (success, warning, error), or an overlay annotation on an image. Keep the text brief; badges are not a substitute for body copy or tooltips.

## Import

```tsx
import { Badge } from '@cuweb/rds2';
```

## Basic usage

```tsx
// Static label
<Badge text="Research" />

// Link badge
<Badge text="Events" href="/events" />

// Semantic state
<Badge text="Open" color="green" rounded="full" />

// Overlay on a dark background
<Badge text="Featured" color="black80" rounded="full" />
```

## CSS classes

| Class | When applied |
|---|---|
| `cu-badge` | Always — base class |
| `cu-badge--{color}` | The active `color` prop value, e.g. `cu-badge--green` |
| `cu-badge--radius-sm` | `rounded="sm"` |
| `cu-badge--radius-md` | `rounded="md"` (default) |
| `cu-badge--radius-lg` | `rounded="lg"` |
| `cu-badge--radius-full` | `rounded="full"` |
| `cu-badge--radius-none` | `rounded="none"` |

When rendered as a link, the `a.cu-badge` selector additionally removes underline decoration at rest and restores it on hover.

## Accessibility

- A badge with no `href` is a `<span>` — it carries no interactive role and is read inline by screen readers. Do not rely on it to convey critical information through color alone; always include meaningful `text`.
- When `href` is provided the badge becomes a focusable link. Ensure the `text` value is descriptive enough to make sense out of context (e.g. "Research" rather than "Click here").
- `black80` and `white80` are semi-transparent. Verify sufficient contrast against whatever background they appear on — the design system cannot guarantee contrast for arbitrary backgrounds.
- Semantic color variants (`green`, `red`, `yellow`, `blue`) reinforce meaning visually but the text itself must still convey the state, since color is invisible to some users.
