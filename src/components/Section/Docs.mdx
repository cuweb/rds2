# Section

A page-level content container that controls width, background, and vertical rhythm. Nests inside `Main` and wraps layout components like `Column`, `Article`, and `Aside`. Three max-width presets align with the WordPress block editor's alignment system.

## When to use

Use `Section` to group related content into a distinct visual band ā€” a card grid, an article body, a call-to-action strip. Stack multiple `Section` components inside `Main` to build alternating white and grey content rows. Use `maxWidth="alignfull"` with `bgType="grey"` for full-bleed accent sections. Add `contentWidth` when you want the section background to span one width while the children are constrained to a narrower readable column.

## Import

```tsx
import { Section } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
// Default ā€” content width, white background
<Main>
  <Section>
    <Column cols="3">...</Column>
  </Section>
</Main>

// Grey background
<Main>
  <Section bgType="grey">
    <Column cols="2">...</Column>
  </Section>
</Main>

// Black background (automatically inverts text to white)
<Main>
  <Section bgType="black">
    <Column cols="2">...</Column>
  </Section>
</Main>

// Light gradient background
<Main>
  <Section bgType="light-gradient">
    <Column cols="2">...</Column>
  </Section>
</Main>

// Full-bleed grey band
<Main>
  <Section maxWidth="alignfull" bgType="grey">
    <Column cols="3">...</Column>
  </Section>
</Main>

// Full-bleed grey band with content constrained to 1024px
<Main>
  <Section maxWidth="alignfull" bgType="grey" contentWidth="aligncontent">
    <Column cols="2">...</Column>
  </Section>
</Main>

// Full-bleed grey band with content constrained to 1280px
<Main>
  <Section maxWidth="alignfull" bgType="grey" contentWidth="alignwide">
    <Column cols="2">...</Column>
  </Section>
</Main>
```

## Max-width presets

| `maxWidth`       | Token                        | Description                                    |
| ---------------- | ---------------------------- | ---------------------------------------------- |
| `"aligncontent"` | `--rds--layout-content-size` | Default ā€” standard readable content width      |
| `"alignwide"`    | `--rds--layout-wide-size`    | Wider ā€” for card grids and media-rich sections |
| `"alignfull"`    | `--rds--layout-full-size`    | Full viewport width ā€” hero and accent bands    |

These names and tokens mirror the WordPress block editor's alignment system so that React templates and WordPress block themes share the same layout vocabulary.

## The `as` prop

By default `Section` renders a `<section>` element. Pass `as="div"` when a section landmark is semantically inappropriate ā€” for example, a purely decorative spacer row or a wrapper that forms part of a larger landmark region.

## The `contentWidth` prop

When set, `contentWidth` wraps all children in a `cu-section__content` div independently constrained within the outer layout:

| Value            | Inner content width                                   |
| ---------------- | ----------------------------------------------------- |
| `"aligncontent"` | 1024px ā€” standard readable content width              |
| `"alignwide"`    | 1280px ā€” wider, for card grids and media-rich layouts |
| `"alignfull"`    | Full viewport width                                   |

Omitting `contentWidth` keeps the existing behaviour: children are direct children of the root and the root's `is-layout-constrained` applies.

## CSS classes

| Class                        | When applied                                          |
| ---------------------------- | ----------------------------------------------------- |
| `cu-section`                 | Always                                                |
| `cu-section--grey`           | `bgType="grey"`                                       |
| `cu-section--black`          | `bgType="black"`                                      |
| `cu-section--light-gradient` | `bgType="light-gradient"`                             |
| `aligncontent`               | `maxWidth="aligncontent"` (default)                   |
| `alignwide`                  | `maxWidth="alignwide"`                                |
| `alignfull`                  | `maxWidth="alignfull"`                                |
| `has-global-padding`         | When `bgType` is set ā€” horizontal gutter              |
| `is-layout-constrained`      | Always ā€” constrains children to content / wide widths |

| `--rds--color-grey-pale`             | Grey section background                         |

| `--rds--layout-full-size`            | `alignfull` max-width (100%)                    |

## Accessibility

- `<section>` is a landmark element when it has an accessible name ā€” add `aria-label` or `aria-labelledby` via the standard HTML prop when the section's purpose should be surfaced to screen reader navigation.
- Without an accessible name, `<section>` is not exposed as a landmark, which is acceptable for generic layout sections. Use `as="div"` if you actively want to suppress any landmark semantics.
- When stacking many sections on a single page, ensure each meaningful `<section>` landmark has a distinct label so screen reader users can distinguish between them in landmark navigation.
