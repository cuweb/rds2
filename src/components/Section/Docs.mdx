# Section

A page-level content container that controls width, background, and vertical rhythm. Nests inside `Main` and wraps layout components like `Column`, `Article`, and `Aside`. Three max-width presets align with the WordPress block editor's alignment system.

## When to use

Use `Section` to group related content into a distinct visual band — a card grid, an article body, a call-to-action strip. Stack multiple `Section` components inside `Main` to build alternating white and grey content rows. Use `maxWidth="alignfull"` with `isGrey` for full-bleed accent sections.

## Import

```tsx
import { Section } from '@cuweb/rds2';
```

## Basic usage

```tsx
// Default — content width, white background
<Main>
  <Section>
    <Column cols="3">...</Column>
  </Section>
</Main>

// Grey background
<Main>
  <Section isGrey>
    <Column cols="2">...</Column>
  </Section>
</Main>

// Full-bleed grey band
<Main>
  <Section maxWidth="alignfull" isGrey>
    <Column cols="3">...</Column>
  </Section>
</Main>
```

## Max-width presets

| `maxWidth` | Token | Description |
|---|---|---|
| `"aligncontent"` | `--rds--layout-content-size` | Default — standard readable content width |
| `"alignwide"` | `--rds--layout-wide-size` | Wider — for card grids and media-rich sections |
| `"alignfull"` | `--rds--layout-full-size` | Full viewport width — hero and accent bands |

These names and tokens mirror the WordPress block editor's alignment system so that React templates and WordPress block themes share the same layout vocabulary.

## The `as` prop

By default `Section` renders a `<section>` element. Pass `as="div"` when a section landmark is semantically inappropriate — for example, a purely decorative spacer row or a wrapper that forms part of a larger landmark region.

## CSS classes

| Class | When applied |
|---|---|
| `cu-section` | Always |
| `cu-section--white` | `isGrey` is false (default) |
| `cu-section--grey` | `isGrey={true}` |
| `aligncontent` | `maxWidth="aligncontent"` (default) |
| `alignwide` | `maxWidth="alignwide"` |
| `alignfull` | `maxWidth="alignfull"` |
| `has-global-padding` | Always — horizontal gutter from `--rds--layout-gutter` |
| `is-layout-constrained` | Always — constrains children to content / wide widths |

## Design tokens

| Token | Used for |
|---|---|
| `--rds--color-grey-pale` | Grey section background |
| `--rds--spacing-large` | Internal padding on grey sections |
| `--rds--radius-xl` | Border radius on non-full-width grey sections |
| `--rds--layout-content-size` | `aligncontent` max-width |
| `--rds--layout-wide-size` | `alignwide` max-width |
| `--rds--layout-full-size` | `alignfull` max-width (100%) |

## Accessibility

- `<section>` is a landmark element when it has an accessible name — add `aria-label` or `aria-labelledby` via the standard HTML prop when the section's purpose should be surfaced to screen reader navigation.
- Without an accessible name, `<section>` is not exposed as a landmark, which is acceptable for generic layout sections. Use `as="div"` if you actively want to suppress any landmark semantics.
- When stacking many sections on a single page, ensure each meaningful `<section>` landmark has a distinct label so screen reader users can distinguish between them in landmark navigation.
