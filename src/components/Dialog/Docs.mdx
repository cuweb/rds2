# Dialog

A blocking confirmation / action prompt. Renders as a native `<dialog>` opened with `showModal()`, with a built-in title, optional description, and a button footer. Unlike `Modal`, it cannot be dismissed via the Esc key or a backdrop click — the user must choose an action button.

## When to use

- Confirm a destructive or hard-to-reverse action (e.g. delete, discard changes)
- Require an explicit acknowledgement before continuing
- Surface a short, single-purpose message with one or more action buttons

For freeform, dismissible overlay content without a fixed title/description/footer shape, use `Modal` instead.

## Import

```tsx
import { Dialog } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
const [isOpen, setIsOpen] = useState(false);

// With a confirm action
<Dialog
  isOpen={isOpen}
  setIsOpen={setIsOpen}
  title="Delete this item?"
  description="This action cannot be undone."
>
  <Button title="Delete" onClick={() => setIsOpen(false)} isSmall />
</Dialog>

// Title only, no description or extra actions
<Dialog isOpen={isOpen} setIsOpen={setIsOpen} title="Your changes have been saved." />
```

## CSS classes

| Class                    | When applied                                       |
| ------------------------ | -------------------------------------------------- |
| `cu-dialog`              | Always — base class, the `<dialog>` element itself |
| `cu-dialog__body`        | Wrapper around the title and description           |
| `cu-dialog__title`       | Title `<h2>` element                               |
| `cu-dialog__description` | Optional description `<p>` element                 |

| `--rds--font-weight-semibold` | Title font weight                  |

## Accessibility

- Renders as a native `<dialog>` opened via `showModal()`, which traps focus within the dialog and restores focus to the triggering element on close.
- `aria-labelledby` points at the title; `aria-describedby` points at the description when one is provided.
- The Esc key and backdrop clicks are intentionally disabled — the dialog can only be closed via an action button, so destructive actions are never dismissed accidentally.
- The built-in "Close" button always renders last in the footer, after any action buttons passed as `children`.
