# SearchInput

A controlled search field component that calls a callback on every keystroke, with an optional `SearchInput.Results` subcomponent for rendering a filtered auto-suggest list.

## When to use

- To provide a live-filtering search experience within a page or dialog
- When a list of results should update as the user types
- As a site-wide search field inside a nav or modal
- When you need a search input that is framework-agnostic (works with Next.js, WordPress, etc.)

## Import

```tsx
import { SearchInput } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
// Simple search input without results
<SearchInput placeholder="Search..." callback={(value) => console.log(value)} />
```

```tsx
// With auto-suggest results
<SearchInput callback={setSearchString} placeholder="Search...">
    <SearchInput.Results resultsData={filteredResults} />
</SearchInput>
```

## CSS classes

| Class                           | Condition                                     |
| ------------------------------- | --------------------------------------------- |
| `cu-search-input`               | Always — root wrapper element                 |
| `cu-search-input__icon`         | Always — magnifying-glass icon                |
| `cu-search-input__field`        | Always — the `<input>` element                |
| `cu-search-input__results`      | When `children` is provided                   |
| `cu-search-input__results-list` | Always on the `<ul>` in `SearchInput.Results` |
| `cu-search-input__results-item` | Applied to each `<li>` result row             |
| `cu-search-input__results-link` | Applied to each result link                   |

## Accessibility

- The `<input>` uses `type="search"` which provides native clear button behaviour in supporting browsers
- The magnifying-glass `<Icon>` is decorative and rendered with `aria-hidden="true"` automatically
- Pair the input with a visible `<label>` or `aria-label` in your consuming page to meet WCAG 2.1 SC 1.3.1 — the component does not render a label internally to remain layout-agnostic
- The results `<ul>` list provides semantic structure; screen readers announce the number of items when focus enters the list
- Result links are standard `<a>` elements and are keyboard-navigable via Tab
