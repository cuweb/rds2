# Footer

A minimal site footer displaying the Carleton University logo above three meta links: Privacy Policy, Accessibility, and a copyright notice with the current year. Use this variant on microsites, campaign pages, or any context where the full link-column footer would be too heavy.

## When to use

- Microsites and campaign landing pages that do not need the full link directory
- Error and maintenance pages
- Any page where a lightweight footer is preferred over [`FooterStandard`](?path=/docs/components-navigation-footer-standard--docs)

## Import

```tsx
import { Footer } from '@cuweb/rds2';
```

## Basic usage

```tsx
// All defaults — Carleton logo and standard meta links
<Footer />

// Custom links
<Footer
  privacyHref="/privacy"
  accessibilityHref="/accessibility"
  copyrightHref="/copyright"
/>
```

## Logo

The logo defaults to the Carleton University colour vertical outlined logo from the built-in asset registry. Pass `logoSrc` to swap in a department or campaign logo, and update `logoAlt` accordingly.

## Copyright year

The copyright year is generated dynamically from `new Date().getFullYear()` — no prop is needed and the year updates automatically at the turn of the year.

## Links via LinkProvider

All three meta links render through the `LinkProvider` context. Wrap the application root with `LinkProvider` to swap the plain `<a>` for a framework-specific `<Link>` component:

```tsx
import Link from 'next/link';
import { LinkProvider, Footer } from '@cuweb/rds2';

<LinkProvider component={Link}>
  <Footer />
</LinkProvider>
```

## CSS classes

`Footer` uses the same stylesheet as `FooterStandard` and shares several classes.

| Class | Element | Notes |
|---|---|---|
| `cu-footer` | `<footer>` | Black background, CU waves image at the bottom |
| `cu-footer--basic` | `<footer>` | Modifier for the minimal footer; `background-size: cover` |
| `cu-footer__logo-links` | Wrapper `<div>` | Centred grid; bottom padding leaves space for the waves |
| `cu-footer__logo` | `<img>` | `8rem` tall on mobile, `11rem` on `sm+` |
| `cu-footer__meta` | `<ul>` | Flex row, centred, wraps on narrow screens |
| `cu-footer__meta-item` | `<li>` | Gap `1rem` mobile / `1.5rem` on `sm+` |
| `cu-footer__meta-link` | `<a>` | `grey-light` colour, no underline; white on hover |

## Design tokens

| Token | Used for |
|---|---|
| `--rds--color-grey-light` | Link and text colour |
| `--rds--color-black` | Footer background |
| `--rds--asset-cu-waves-footer-red` | Decorative waves background image |
| `--rds--layout-wide-size` | Inner max-width (via `FooterStandard` styles) |
| `--rds--font-size-x-small` | Meta link font size on mobile |
| `--rds--font-size-small` | Meta link font size on `sm+` |

## Accessibility

- `<footer>` is a landmark element. It is announced as a contentinfo landmark by screen readers when it is a direct descendant of `<body>`.
- A visually hidden `<h2 id="cu-footer-heading">Footer</h2>` is present and referenced via `aria-labelledby` on `<footer>` — this gives the landmark an accessible name in all browsers.
- All meta links are native anchors and are fully keyboard accessible.
