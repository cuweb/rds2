# Nav

A sticky site header built from composable sub-components. Six named pieces ‚Äî `Nav.Top`, `Nav.Bottom`, `Nav.Logo`, `Nav.Menu`, `Nav.Buttons` ‚Äî assemble into two distinct layouts depending on whether the site has a department identity. The menu implements priority-plus overflow so items are never truncated or wrapped; any that do not fit are collected into a "Browse" dropdown automatically.

## Import

```tsx
import { Nav } from '@cuweb/rds2';
```

## Compositions

`Nav` supports two layout patterns. Choose based on whether the site needs a department identity row separate from the main navigation.

### Top-only

Logo, menu, and buttons share one row. Use this for the main Carleton site or any site without a secondary row.

```tsx
<Nav>
  <Nav.Top>
    <Nav.Logo />
    <Nav.Menu menu={navData} />
    <Nav.Buttons
      isSearch
      buttons={[
        { title: 'Apply', href: '/apply' },
        { title: 'Donate', href: '/donate', variant: 'dark' },
      ]}
    />
  </Nav.Top>
</Nav>
```

### Top + Bottom

Logo and CTA buttons sit in the top row; the menu moves to a grey accent strip below. Use this for faculty and department sites where the site title and CTAs should stay visible while the nav scrolls.

```tsx
<Nav>
  <Nav.Top>
    <Nav.Logo title="Faculty of Arts and Social Sciences" link="/arts" />
    <Nav.Buttons buttons={[{ title: 'Apply', href: '/apply' }]} />
  </Nav.Top>
  <Nav.Bottom>
    <Nav.Menu menu={navData} />
  </Nav.Bottom>
</Nav>
```

## Sub-components

### Nav.Logo

| Prop | Type | Default | Notes |
|---|---|---|---|
| `title` | `string` | ‚Äî | Department or site name; renders as a linked text label |
| `link` | `string` | ‚Äî | `href` for the site title link (requires `title`) |

When `title` is provided the horizontal CU wordmark is hidden on mobile and replaced with the CU shield so the site title has room. Both images are always in the DOM ‚Äî the swap is CSS-driven.

The Carleton University logomark always links to `https://carleton.ca`. The site title link uses `LinkProvider` context and adopts any injected router component.

### Nav.Menu

| Prop | Type | Notes |
|---|---|---|
| `menu` | `NavItem[]` | The full navigation tree |

**Priority-plus overflow:** On mount and on every container resize, `Nav.Menu` measures a hidden off-flow clone of all items to find how many fit within the available width. Items that fit are shown inline; the remainder are collected under a "Browse" button. The Browse button is always in the DOM (ghost state) so its width is always measurable ‚Äî it becomes interactive only when there is overflow.

**Submenus:** Items with a `submenu` array render as toggle buttons rather than links. Clicking opens a dropdown panel. Submenus support one level of nesting inside the dropdown (two levels total: top-level ‚Üí submenu ‚Üí nested submenu).

**Mobile (`< 600px`):** All inline items are hidden; only the Browse trigger is shown. The entire menu tree is accessible via Browse. Scrolling the browse panel is capped at the viewport height.

### Nav.Buttons

| Prop | Type | Default | Notes |
|---|---|---|---|
| `buttons` | `NavButton[]` | ‚Äî | Array of CTA links |
| `isSearch` | `boolean` | `false` | Shows a search icon button |
| `onClickSearch` | `() => void` | ‚Äî | Handler called when the search button is clicked |

CTA links render through `LinkProvider` and adopt any injected router component. The `variant` on each `NavButton` controls colour ‚Äî `'primary'` (default) is red, `'dark'` is black.

```ts
interface NavButton {
  title: string;
  href: string;
  variant?: 'primary' | 'dark';
}
```

Returns `null` if neither `isSearch` nor any `buttons` are provided.

## NavItem type

```ts
interface NavItem {
  title: string;
  href?: string;       // leaf items need an href; parent items can omit it
  submenu?: NavItem[]; // one level of nesting is supported inside dropdowns
}
```

## Scroll behaviour

`Nav` is `position: sticky; top: 0` ‚Äî it stays pinned as the user scrolls. On scroll **down**, the top row (white logo strip) slides smoothly off-screen, leaving only the bottom strip (or the grey bar on mobile) pinned at the top. On scroll **up**, the logo row returns. A fixed 2 px red accent line sits above the header at all times and never moves.

The shift amount is calculated from the DOM rather than hardcoded ‚Äî it measures the bottom row's position relative to the header so the pin point is always exact, regardless of logo height or padding changes.

## Mobile layout

Below the `sm` breakpoint (600 px):

- **Top-only composition:** The menu row drops to a grey bar below the logo. The logo spans the full width. CTA buttons sit to the right of the Browse trigger on the grey bar.
- **Top + Bottom composition:** The menu stays in `Nav.Bottom`. The `Nav` wrapper detects this pattern and moves any CTA buttons from `Nav.Top` into `Nav.Bottom` at runtime so they sit beside Browse, matching the top-only mobile layout.

## Accessibility

- The outer `<header>` is a banner landmark. The inner `<nav aria-label="Site navigation">` is a navigation landmark ‚Äî screen reader users can jump to it directly.
- All toggle buttons (`Nav.Menu` items with submenus, Browse) carry `aria-expanded` so screen readers report open/closed state. The Browse button's `aria-expanded` and `aria-haspopup` are omitted in ghost state (when no overflow exists) to avoid announcing a non-functional control.
- The ghost Browse button uses `aria-hidden` and `tabIndex={-1}` when there is no overflow so it is not reachable by keyboard.
- Dropdowns close on `Escape` and on outside `mousedown` ‚Äî keyboard users can always dismiss with Escape.
- The search button has `aria-label="Search"` and its icon is `aria-hidden`.
- Social and CTA links rendered through `LinkProvider` are native anchors and fully keyboard accessible.
- The measurement clone list is `aria-hidden="true"` and `pointer-events: none` so it is invisible to assistive technology and mouse.
- Mobile body scroll lock (`overflow: hidden` on `<body>`) is applied only while the Browse panel is open on mobile, preventing background scroll while the panel is in view.
