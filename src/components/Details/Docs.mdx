# Details

A structured metadata list for displaying concise factual information — event details, contact info, programme attributes, and similar content — as an accessible `<ul>` or `<ol>`.

## When to use

- Displaying key details for an event (date, location, cost, link)
- Listing contact information for a person or department
- Showing programme or course metadata in a scannable format
- Pairing factual items with icons to aid recognition at a glance

## Import

```tsx
import { Details } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
<Details>
    <Details.Item isBold>Hybrid Event</Details.Item>
    <Details.Item>Raven's Nest, 1125 Colonel By Drive</Details.Item>
    <Details.Item>
        <a href="/teams">Teams meeting link</a>
    </Details.Item>
    <Details.Item iconName="tag">
        <strong>Cost:</strong> $20 per adult, $15 for youth/senior
    </Details.Item>
</Details>
```

With dividers and a grey background:

```tsx
<Details hasDividers isGrey>
    <Details.Item isBold>Contact</Details.Item>
    <Details.Item>
        Jane Doe, <a href="mailto:jane@example.com">Email Jane</a>, 613-520-2600 x1234
    </Details.Item>
</Details>
```

## CSS classes

| Class                    | Condition                                     |
| ------------------------ | --------------------------------------------- |
| `cu-details`             | Always — applied to the root `<ul>` or `<ol>` |
| `cu-details--dividers`   | When `hasDividers` is `true`                  |
| `cu-details--grey`       | When `isGrey` is `true`                       |
| `cu-details__item`       | Always — applied to each `<li>`               |
| `cu-details__item--bold` | When `isBold` is `true` on `Details.Item`     |
| `cu-details__item--icon` | When `iconName` is provided on `Details.Item` |

## Accessibility

- Renders as a semantic `<ul>` by default; pass `as="ol"` when order is meaningful
- List markers are hidden visually but the list semantics are preserved for screen readers — items are announced as list members with a count
- Links within items inherit the component's link colour tokens and include underline on hover; avoid using colour as the sole indicator of interactivity
- Icons are decorative by default (`aria-hidden` is set on the `<Icon>` component); if an icon is the only content conveying meaning, add a visible text label alongside it
- `isBold` is a visual cue only — do not rely on it alone to convey semantic importance; wrap text in `<strong>` inside the item for screen reader emphasis when needed
