# Aside

A semantic `<aside>` wrapper for supplementary content — sidebars, related links, callout boxes, or any content complementary to the main article. Supports optional sticky positioning via the `isSticky` prop.

## When to use

Use `Aside` inside a `Column` layout alongside a main content area. The most common patterns are a right-hand sidebar (navigation, related content) or a left-hand sticky sidebar (table of contents, filters).

## Import

```tsx
import { Aside } from '@cuweb/rds2';
```

## Basic usage

```tsx
// Standard sidebar
<Column cols="2/3">
  <Column.Content>
    <p>Main content...</p>
  </Column.Content>
  <Aside>
    <p>Related links...</p>
  </Aside>
</Column>

// Sticky left sidebar
<Column cols="1/3">
  <Aside isSticky topSpace={80}>
    <nav>Table of contents</nav>
  </Aside>
  <Column.Content>
    <p>Long-form content...</p>
  </Column.Content>
</Column>
```

## Sticky positioning

When `isSticky` is `true`, the component wraps children in `<div class="sticky" style={{ top: topSpace + 'px' }}>`. The `sticky` class is expected to provide `position: sticky` — either via a utility CSS framework in the consumer's project or a future base style addition to RDS.

> **Note:** The `cu-aside` and `cu-prose` classes applied to the `<aside>` element, along with `relative` and `sticky` on the inner wrapper, are not currently defined in the RDS stylesheet. Styling from these class names must come from the consumer project.

## CSS classes

| Class | Element | Notes |
|---|---|---|
| `cu-aside` | `<aside>` | Always — RDS hook, not yet styled internally |
| `cu-prose` | `<aside>` | Always — prose typography hook, not yet styled internally |
| `relative` | `<aside>` | Always — expected to provide `position: relative` |
| `sticky` | Inner `<div>` | Only when `isSticky` is true — expected to provide `position: sticky` |

## Accessibility

- `<aside>` is a complementary landmark. Screen readers surface it as a navigable region distinct from the main content.
- The `landmark-complementary-is-top-level` axe rule is suppressed in the Storybook stories because `<aside>` is nested inside `<main>` for demo purposes. In production, this nesting is valid and the rule can be configured per project.
- Ensure the aside has a discernible label when multiple complementary landmarks appear on the same page — add `aria-label` or `aria-labelledby` via the `...rest` passthrough if needed.
