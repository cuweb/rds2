# Footer Standard

A full-featured site footer with four regions: an Indigenous land acknowledgment, a multi-column link directory, a contact and social links row, and a logo with meta links. Three pre-configured link sets are available via the `type` prop — `standard`, `athletics`, and `futureFunder`.

## When to use

Use `FooterStandard` on all full-site pages. For lightweight contexts (microsites, campaign pages) use [`Footer`](?path=/docs/components-navigation-footer--docs) instead.

## Import

```tsx
import { FooterStandard } from '@cuweb/rds2';
```

## Basic usage

```tsx
// Standard Carleton footer — all defaults
<FooterStandard />

// Athletics site footer
<FooterStandard type="athletics" />

// Custom acknowledgment text
<FooterStandard acknowledgment="Custom land acknowledgment text here." />
```

## Footer regions

### Acknowledgment

Full-width centred text above the link columns. Defaults to Carleton's institutional land acknowledgment. Pass `acknowledgment` to override.

### Link columns

Driven entirely by data — the `type` prop selects a pre-built set of `FooterLinkColumn[]` from `FooterData.ts`. Each column contains one or more `FooterLinkGroup` objects with a heading, a list of links, and an optional media item (image, optionally linked).

The columns region is hidden on mobile and visible on `md+` screens.

### Contact and social

Phone and email links render conditionally — if `phoneHref` or `contactHref` is absent from the `contact` object the corresponding text fragment is suppressed. The address line is always shown.

Social icons are rendered using the `Icon` component and pull from the `social` array. Defaults include Facebook, Instagram, X (Twitter), YouTube, and LinkedIn.

### Logo and meta links

Identical to the [`Footer`](?path=/docs/components-navigation-footer--docs) component — Carleton logo above Privacy Policy, Accessibility, and © Copyright links. The copyright year is generated dynamically.

## Customising contact info

```tsx
<FooterStandard
  contact={{
    phone: '613-555-0100',
    phoneHref: 'tel:+1-613-555-0100',
    contactHref: '/contact',
    address: '1 Example Street, Ottawa, ON, K1A 0A0',
  }}
/>
```

## Customising social links

```tsx
import type { FooterSocialLink } from '@cuweb/rds2/data/FooterData';

const social: FooterSocialLink[] = [
  { name: 'Facebook', href: 'https://facebook.com/mypage', icon: 'facebook' },
  { name: 'Instagram', href: 'https://instagram.com/mypage', icon: 'instagram' },
];

<FooterStandard social={social} />
```

## Links via LinkProvider

All links in `FooterStandard` — column links, contact links, social links, and meta links — render through the `LinkProvider` context. Wrap the app root with `LinkProvider` once and every link adopts the injected component automatically.

## Accessibility

- `<footer>` is a contentinfo landmark, announced as such when it is a top-level descendant of `<body>`.
- A visually hidden `<h2>Footer</h2>` is referenced by `aria-labelledby` on `<footer>` — it gives the landmark an accessible name across all browsers.
- Column headings are `<h3>` elements — ensure the page heading hierarchy above `FooterStandard` does not skip from `<h1>` to `<h3>`.
- Social icon links are visually icon-only; each has a `<span class="sr-only">` with the platform name to provide an accessible label.
- All links render through `LinkProvider` and remain keyboard accessible regardless of which link component is injected.
