# Cookie Banner

A fixed-position consent notice that appears at the bottom of the viewport until the user accepts. Visibility is controlled by a browser cookie — once accepted the banner stays hidden for 365 days. All text content is customisable; the defaults are pre-filled with Carleton's privacy policy URL.

## When to use

Place `CookieBanner` once at the application root, outside of any scrollable content. It renders nothing until the consent cookie is absent or expired, so it is safe to include unconditionally on every page.

## Import

```tsx
import { CookieBanner } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
// Default — Carleton privacy policy text and URL
<CookieBanner />

// Custom text and policy link
<CookieBanner
  message="We use cookies to improve your experience. Learn more about"
  policyHref="/privacy"
  policyLabel="our privacy policy."
  buttonLabel="Accept"
/>
```

## How visibility works

On mount, the component reads `document.cookie` for an entry matching `cookieName`. If no entry exists — or if the browser has already expired it — the banner is shown. Clicking the accept button calls `setCookie`, which writes a cookie that expires in 365 days via the browser's native `expires` attribute, and immediately hides the banner via local state.

The component is SSR-safe: both `isCookieExpired` and `setCookie` guard against `typeof document === 'undefined'` and return safe defaults on the server.

## Cookie utilities

Two helpers from `cookies.ts` are used internally and can be imported directly if needed:

```ts
import { isCookieExpired, setCookie } from '@cuweb/raven-design-system/CookieBanner';

// Returns true if the cookie is absent (banner should show)
isCookieExpired('cookieConsent');

// Writes the cookie for the given number of days (default 365)
setCookie('cookieConsent', 365);
```

## Accessibility

- The banner renders with `role="dialog"` and `aria-label="Cookie notice"` — screen readers announce it as a modal dialog when it appears.
- `aria-live="polite"` ensures the notice is announced without interrupting content already being read.
- The policy link renders through the `LinkProvider` context, so it adopts any framework-specific `<Link>` component injected at the app root.
- The accept button uses the native `<Button>` component — keyboard accessible and announced as a button.
- When the banner is dismissed (`isVisible` becomes `false`) the component returns `null`, removing it from the DOM entirely rather than hiding it with CSS — the dialog disappears cleanly for both sighted and assistive technology users.
