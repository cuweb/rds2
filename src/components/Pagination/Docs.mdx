# Pagination

A fully-controlled navigation component that renders a row of page buttons with previous/next controls and smart ellipsis for large page counts.

## When to use

- Navigating through a paginated list of content such as news articles, search results, or events
- Anywhere a large data set is split into discrete pages and the caller manages the active page in state
- As a companion to card grids or listing layouts where only a subset of items is shown at once
- When a URL-driven or state-driven page index needs a visual control

## Import

```tsx
import { Pagination } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
const [page, setPage] = useState(1);

<Pagination currentPage={page} totalPages={8} onPageChange={setPage} />;
```

## Accessibility

- The component renders a `<nav>` element — a landmark region. The `ariaLabel` prop (default `"Pagination"`) is applied as `aria-label` so screen readers can distinguish it from other navigation landmarks on the page.
- Page buttons are wrapped in an `<ol>` with `<li>` items, conveying list semantics.
- The active page button carries `aria-current="page"`, which screen readers announce as the current location.
- Previous and Next buttons receive `disabled` and `aria-disabled="true"` when the user is at the first or last page respectively, preventing interaction and communicating the boundary to assistive technology.
- Chevron icons inside Prev/Next are decorative and carry `aria-hidden="true"`; each button has an explicit `aria-label` (`"Previous page"` / `"Next page"`) so the action is announced without relying on the icon.
- Ellipsis spans carry `aria-hidden="true"` — they are visual punctuation only and add no useful information for screen reader users.
- All interactive buttons expose a visible `:focus-visible` outline styled with the primary colour token, meeting WCAG 2.4.7 (Focus Visible).
