# Button

A flexible action trigger that renders a native `<button>` element. Supports five color variants, optional Font Awesome icons, size and width modifiers, and a disabled state.

## When to use

Use `Button` when you need an interactive control that triggers an in-page action — submitting a form, opening a dialog, or kicking off a process. For links that navigate to a URL, prefer a semantic `<a>` element or your framework's `Link` component instead. When placing two or more buttons side by side, wrap them in a [`ButtonGroup`](?path=/docs/components-elements-button-group--docs) rather than managing spacing manually.

## Import

```tsx
import { Button } from '@cuweb/raven-design-system';
```

## Basic usage

```tsx
<Button title="Get started" color="red" />
<Button title="Download" color="dark-grey" icon="circle-down" />
<Button icon="circle-down" ariaLabel="Download file" color="dark-grey" />
```

## CSS classes

| Class                  | When applied            |
| ---------------------- | ----------------------- |
| `cu-button`            | Always — base class     |
| `cu-button--red`       | `color="red"` (default) |
| `cu-button--grey`      | `color="grey"`          |
| `cu-button--dark-grey` | `color="dark-grey"`     |
| `cu-button--black`     | `color="black"`         |
| `cu-button--white`     | `color="white"`         |
| `cu-button--disabled`  | `isDisabled={true}`     |
| `cu-button--small`     | `isSmall={true}`        |
| `cu-button--full`      | `isFull={true}`         |

A top margin (`2rem` mobile, `2.5rem` desktop) is automatically applied when a `Button` or `ButtonGroup` directly follows a `Column` or `StackedList` element — no extra spacing class needed.

## Design tokens

These `--rds--*` custom properties drive the Button's appearance. Overriding them at a higher scope will change every Button on the page.

| Token                       | Used for                                                                        |
| --------------------------- | ------------------------------------------------------------------------------- |
| `--rds--font-family-inter`  | Font family                                                                     |
| `--rds--font-size-small`    | Font size on mobile and when `isSmall`                                          |
| `--rds--font-size-medium`   | Font size on desktop (≥ md breakpoint)                                          |
| `--rds--font-weight-medium` | Font weight                                                                     |
| `--rds--line-height-tight`  | Line height                                                                     |
| `--rds--radius-md`          | Border radius                                                                   |
| `--rds--color-primary`      | `red` variant background; hover state for `grey`, `dark-grey`, `black`, `white` |
| `--rds--color-grey-dark`    | `dark-grey` variant background; `grey` variant text                             |
| `--rds--color-grey-pale`    | `grey` variant background                                                       |
| `--rds--color-grey-light`   | Disabled state background                                                       |
| `--rds--color-white`        | `white` variant background; text on dark variants                               |
| `--rds--color-black`        | `black` variant background; `white` variant text                                |

## Accessibility

- `Button` renders a native `<button>` element — keyboard focus, `Enter`/`Space` activation, and screen reader announcement are built in.
- **Icon-only buttons require `ariaLabel`.** Without it, screen readers announce only the icon name (or nothing), which is not meaningful. The TypeScript union type enforces this at compile time.
- The native `disabled` attribute prevents the button from receiving focus or firing events. Do not use CSS-only disabled styling, as it leaves the button keyboard-reachable.
- Avoid using `Button` for navigation — screen readers expose `role="button"` to users, who expect it to perform an action rather than navigate.
