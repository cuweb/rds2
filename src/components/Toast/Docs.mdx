# Toast

A transient notification card that appears in a fixed overlay region. Renders via a compound component: `Toast.Toaster` positions the region, and each `Toast` card displays an icon, title, body, and a dismiss button.

## When to use

- Confirm a background action that completed (file saved, email sent)
- Report a non-blocking error or warning that the user should acknowledge
- Surface real-time status updates without interrupting the user's flow

Do not use Toast for critical errors that require user action — use `Alert` inline instead.

## Import

```tsx
import { Toast } from '@cuweb/rds2';
```

## Basic usage

```tsx
<Toast.Toaster>
  <Toast type="success">
    <Toast.Title>Changes saved</Toast.Title>
    <Toast.Content>Your profile has been updated successfully.</Toast.Content>
  </Toast>
</Toast.Toaster>
```

## Compound components

| Component | Purpose |
|---|---|
| `Toast.Toaster` | Fixed-position `aria-live` region that stacks notifications |
| `Toast` | Individual notification card with dismiss functionality |
| `Toast.Title` | Title line (semibold, dark) |
| `Toast.Content` | Optional body line (regular, muted) |

## Accessibility

- `Toast.Toaster` uses `aria-live="assertive"` and `aria-atomic="true"` so screen readers announce new toast content immediately.
- Each `Toast` carries `role="status"` — a live region role appropriate for transient notifications.
- The dismiss button has `aria-label="Dismiss notification"` so its purpose is clear without a visible text label.
- Icon is decorative — `aria-hidden="true"` prevents it from being read aloud.
- The dismiss button is keyboard-focusable and has a visible `:focus-visible` ring styled with `--rds--color-primary`.
- Toast type (success, error, etc.) is conveyed through the title text and icon; color is a secondary reinforcement only.
