## Two-package model

`@cuweb/rds-2.0` ships with `@cuweb/rds-icons` as a peer dependency. Both packages must be installed.

- `@cuweb/rds-2.0` (this package) — components, tokens, styles
- `@cuweb/rds-icons` — Font Awesome Pro icon components, distributed as a private package on GitHub Packages

The second package being private is what enforces FA Pro license compliance — anyone can install rds-2.0, but the `<Icon>` component won't resolve without rds-icons present at install time. See the [icons architecture rationale](https://github.com/cuweb/rds-2.0/blob/main/docs/contributing/architecture.md#two-package-icon-model) for the full story.

## Quick start

```sh
# 1. One-time per machine: GitHub Packages auth
#    See: https://github.com/cuweb/rds-icons#developer-onboarding-one-time

# 2. In your project
echo "@cuweb:registry=https://npm.pkg.github.com" >> .npmrc
pnpm add @cuweb/rds-2.0 @cuweb/rds-icons
```

```tsx
import { Button, Icon } from '@cuweb/rds-2.0';
import '@cuweb/rds-2.0/styles';

<Button title="Save" icon="floppy-disk" />
```

For the full install flow with troubleshooting, see [Installation](?path=/docs/overview-getting-started-installation--docs).
