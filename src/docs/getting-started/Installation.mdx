# Installation

How to install `@cuweb/raven-design-system` in any consumer project.

## Prerequisites â€” GitHub Packages auth

`@cuweb/raven-design-system` is private; without auth, `pnpm install` fails with `ERR_PNPM_FETCH_401`. Set up auth once per machine:

```sh
gh auth refresh --scopes read:packages
echo "//npm.pkg.github.com/:_authToken=$(gh auth token)" >> ~/.npmrc
```

Or create a classic PAT at https://github.com/settings/tokens with `read:packages` scope and add it to `~/.npmrc`.

For CI, set the `NODE_AUTH_TOKEN` environment variable to your token.

## 1. Add the scope config

Create or edit `.npmrc` at your project root:

```
@cuweb:registry=https://npm.pkg.github.com
```

Commit this file. It tells pnpm/npm to look for `@cuweb/*` packages on GitHub Packages, not the public npm registry. It does **not** contain any secrets.

## 2. Install the package

```sh
pnpm add @cuweb/raven-design-system
```

## 3. Import styles

Once, near the top of your application:

```tsx
import '@cuweb/raven-design-system/styles';
```

This imports the bundled stylesheet â€” design tokens, base element styles, and every component's CSS.

## 4. Use components

```tsx
import { Badge, Button, Icon } from '@cuweb/raven-design-system';

<Badge text="New" color="green" />
<Button title="Get started" icon="circle-check" />
<Icon name="calendar-clock" size={24} />
```

For framework-specific setup, see [Next.js](?path=/docs/overview-getting-started-next-js--docs) or [WordPress](?path=/docs/overview-getting-started-wordpress--docs).

## Per-component CSS (optional)

If you don't want the full bundled stylesheet, import individual component styles:

```tsx
import { Badge } from '@cuweb/raven-design-system';
import '@cuweb/raven-design-system/Badge.css';
```

This is most useful inside WordPress block bundles. See the full pattern at [`docs/consumers/per-component-css.md`](https://github.com/cuweb/raven-design-system/blob/main/docs/consumers/per-component-css.md).

## Troubleshooting

| Error                                              | Fix                                                                        |
| -------------------------------------------------- | -------------------------------------------------------------------------- |
| `ERR_PNPM_FETCH_401 Unauthorized`                  | Auth not set up, or token expired â€” regenerate your GitHub Packages token  |
| `ERR_PNPM_FETCH_404 Not Found` on `@cuweb/raven-*` | Project's `.npmrc` is missing `@cuweb:registry=https://npm.pkg.github.com` |
