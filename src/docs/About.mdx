# About Raven Design System

Raven Design System (RDS) 2.0 is Carleton University's React component library and design system — a shared visual language that brings consistency to web experiences across campus. The successor to [`cuweb/rds`](https://github.com/cuweb/rds), it pairs design tokens with reusable components so teams can build interfaces that feel unmistakably Carleton while reducing duplication.

## Purpose

University websites serve diverse audiences — prospective students, current students, faculty, alumni, and the broader community. RDS exists so that no matter which team is building the experience, the result is polished, accessible, and on-brand.

## What's shipped today

RDS 2.0 is **early-stage**. The token pipeline, build system, and accessibility tooling are production-ready; the component catalogue is intentionally small and growing. See [Getting Started](?path=/docs/overview-getting-started-overview--docs) for installation.

### Design tokens

A single set of tokens drives the visual language for both the React library and the WordPress block theme. Tokens are defined in [`c2b.config.json`](https://github.com/cuweb/rds-2.0/blob/main/c2b.config.json) and generated by [`@troychaplin/component2block`](https://www.npmjs.com/package/@troychaplin/component2block) into CSS custom properties (`--rds--*`), base element styles, and `theme.json`.

- **Colors** — primary (with dark variant), secondary, semantic states (success, warning, error, info), full neutral scale
- **Typography** — Inter typeface, fluid heading and body scales, four font weights, three line-height options
- **Spacing** — seven-step fluid scale from `2x-small` to `2x-large`
- **Effects** — shadow presets (natural, sharp, outlined), border radii, gradients
- **Layout** — content max-width (1024px), wide max-width (1280px)

Browse the live token reference in the **Stylebook** sidebar entries.

### Components

Sixteen components today, grouped to match the sidebar:

- **Elements** — `Badge`, `BadgeGroup`, `Button`, `ButtonGroup`, `Icon`, `DepartmentBar`
- **Layout** — `Article`, `Aside`, `Body`, `Column`, `Main`, `Section`, `Footer`, `FooterStandard`
- **Utilities** — `LinkProvider` (context for swapping in framework-specific Link components), `CookieBanner`

Each component lives in its own folder under [`src/components/`](https://github.com/cuweb/rds-2.0/tree/main/src/components) with co-located stories, styles, and types. The catalogue grows steadily — see [the changelog](?path=/docs/overview-changelog--docs) for what's been added.

### WordPress integration

`@troychaplin/component2block` outputs a `theme.json` and integration helper alongside the React build, so the WordPress Site Editor and frontend stay in sync with the design system. The token pipeline produces:

- A `theme.json` defining the color palette, gradients, fluid typography, spacing scale, and shadow presets
- Per-block CSS that can be enqueued only when a block is used
- Editor styles so authors see the same token-driven styles while editing

Integration patterns for blocks (static and dynamic), block themes, hybrid themes, and Timber/Twig live in the [WordPress integration guide](https://github.com/cuweb/rds-2.0/tree/main/docs/consumers/wordpress).

### Accessibility tested

Every Storybook story runs through [axe](https://github.com/dequelabs/axe-core) at the `error` threshold — a violation in any story fails CI. See the [accessibility page](?path=/docs/overview-getting-started-accessibility--docs) for the testing approach.

## Design principles

- **Consistency** — a unified look and feel across all Carleton web properties
- **Accessibility** — WCAG-aware components, automated axe checks on every story
- **Composability** — small, focused components that combine into complex interfaces
- **Token-first** — all visual values come from `--rds--*` custom properties; no hardcoded hex codes, font stacks, or pixel values

## Tech stack

- React 18, TypeScript 6
- Vite library mode (ESM + CJS)
- SCSS + CSS custom properties (no Tailwind, no CSS-in-JS)
- Storybook 10 with `@storybook/react-vite`
- Vitest + Playwright for unit and a11y testing
- Distributed as `@cuweb/rds-2.0` on GitHub Packages, paired with the private `@cuweb/rds-icons` peer dependency for Font Awesome Pro icons
